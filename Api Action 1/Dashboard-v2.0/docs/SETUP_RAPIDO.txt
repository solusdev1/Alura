# CONFIGURACAO RAPIDA - MongoDB Atlas

## PASSO 1: Copiar Connection String

No VS Code:
1. Clique no icone da extensao MongoDB na barra lateral esquerda
2. Encontre sua conexao (deve aparecer algo como "Cluster0" ou similar)
3. Clique com o botao DIREITO na conexao
4. Selecione "Copy Connection String"

## PASSO 2: Criar arquivo .env

1. Abra o PowerShell no Dashboard-v2.0
2. Execute:

```powershell
# Criar arquivo .env a partir do exemplo
Copy-Item .env.example .env

# Abrir para editar
notepad .env
```

## PASSO 3: Colar Connection String

No arquivo .env que abriu, cole assim:

```env
MONGODB_URI=sua-connection-string-aqui
MONGODB_DATABASE=action1_inventory
```

Exemplo real:
```env
MONGODB_URI=mongodb+srv://usuario:senha@cluster0.ab1cd.mongodb.net/?retryWrites=true&w=majority
MONGODB_DATABASE=action1_inventory
```

**IMPORTANTE**: Substitua:
- `usuario` pelo seu usuario
- `senha` pela sua senha
- `cluster0.ab1cd` pelo seu cluster real

## PASSO 4: Salvar e Testar

1. Salve o arquivo .env (Ctrl+S)
2. Feche o Notepad
3. Inicie o servidor:

```powershell
cd ..
npm start
```

## RESULTADO ESPERADO

Voce deve ver:

```
[OK] Conectado ao MongoDB Atlas (Nuvem)
Database: action1_inventory
Usando MongoDB como banco de dados
Servidor v2.0 rodando em http://localhost:3002
```

## SE DER ERRO

### "bad auth" ou "authentication failed"
- Verifique usuario e senha no .env
- Certifique-se que nao ha espacos extras

### "network timeout"
- Verifique sua internet
- No MongoDB Atlas (site), adicione seu IP em Network Access

### Ainda usando JSON
- Verifique se o arquivo e .env (NAO .env.example)
- Reinicie o servidor

## VERIFICAR SE FUNCIONOU

1. Acesse: http://localhost:3002/test
2. Clique em "Sincronizar Agora"
3. No VS Code, na extensao MongoDB:
   - Expanda sua conexao
   - Expanda "action1_inventory"
   - Veja as collections "devices" e "metadata"

PRONTO! Seus dados estao na nuvem!
